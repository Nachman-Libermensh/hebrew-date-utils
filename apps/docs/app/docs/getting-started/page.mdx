# Getting Started

## Core-first structure

This monorepo contains two libraries and one docs app:

- `packages/hebrew-date-utils` - the main Core package (framework-agnostic)
- `packages/hebrew-date-picker-react` - optional React UI package built on top of the core
- `apps/docs` - documentation site

## Install only the Core package (recommended start)

Use `hebrew-date-utils` in any Node.js project, even without React.

```bash
npm install hebrew-date-utils
```

```ts
import {
  toDualDate,
  listDaysInHebrewMonth,
  getHolidaysOn,
  differenceInDualDays,
} from "hebrew-date-utils";

const dual = toDualDate(new Date());
const monthDays = listDaysInHebrewMonth(dual.hebYear, dual.hebMonth);
const holidaysToday = getHolidaysOn(new Date(), { locale: "he" });

console.log({
  today: dual,
  monthLength: monthDays.length,
  holidaysToday,
  daysFromRoshHashana: differenceInDualDays(new Date(), monthDays[0]!.greg),
});
```

## Optional React package

If your project is React-based and needs a UI picker, add:

```bash
npm install hebrew-date-picker-react hebrew-date-utils
```

The picker package is intentionally secondary: data logic stays in the Core package.

## Install Dependencies

For this monorepo only:

```bash
pnpm install
```

## Build All Packages

```bash
pnpm build
```

## Run Documentation Locally

```bash
pnpm dev:docs
```

## Publish Packages

Publish the core library from `packages/hebrew-date-utils`.

Publish the optional UI package from `packages/hebrew-date-picker-react` when needed.

## GitHub Pages Deployment

Documentation is built as a static export using Next.js:

- `output: "export"`
- `trailingSlash: true`
- `images.unoptimized: true`

The workflow in `.github/workflows/docs-pages.yml` builds `apps/docs` and deploys the `out` directory to GitHub Pages.

Repository package manager: `pnpm` (workspace config in `pnpm-workspace.yaml`).
